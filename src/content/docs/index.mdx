# Q-Store Database Architecture

Q-Store is a novel database architecture that leverages quantum mechanical properties—**superposition**, **entanglement**, **decoherence**, and **tunneling**—as core features rather than obstacles.

## Key Innovation

Instead of fighting quantum mechanics, we design the data infrastructure to **exploit quantum properties** for performance gains impossible with classical systems.

## Quick Example

```python
from quantum_db import QuantumDatabase

# Initialize database
db = QuantumDatabase(
    classical_backend='pinecone',
    quantum_backend='ionq',
    api_key=IONQ_API_KEY
)

# Insert with superposition (multiple contexts)
db.insert(
    id='doc_123',
    vector=embedding,
    contexts=[
        ('normal_query', 0.7),
        ('technical_query', 0.2),
        ('historical_query', 0.1)
    ]
)

# Query with quantum advantages
results = db.query(
    vector=query_embedding,
    context='technical_query',
    enable_tunneling=True,
    top_k=10
)
```

## ML Training Example (v3.3 - NEW)

```python
from q_store.core import QuantumTrainer, QuantumModel, TrainingConfig

# Configure with v3.3 optimizations
config = TrainingConfig(
    quantum_sdk="cirq",
    learning_rate=0.01,
    epochs=5,
    n_qubits=8,
    gradient_method='adaptive',      # 48x fewer circuits
    enable_circuit_cache=True,       # 2-5x speedup
    enable_batch_execution=True      # 5-10x speedup
)

# Train quantum model
trainer = QuantumTrainer(config, backend_manager)
model = QuantumModel(input_dim=8, output_dim=2, n_layers=2, backend=backend)

# Complete training in ~8 minutes (vs 40 minutes in v3.2)
history = await trainer.train(model, data_loader)
```
